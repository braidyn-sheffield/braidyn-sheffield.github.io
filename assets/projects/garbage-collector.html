Simulated Garbage Collector

Home

This project simulates a Java-style memory manager in C using multiple versions of a custom memory allocator, duMalloc. Over four iterations, features like managed pointers, young generation heaps, and minor/major garbage collection were implemented.

Each version incrementally adds memory management complexity to mimic behavior found in high-level managed environments like the JVM. This project demonstrates dynamic memory allocation, garbage collection, and systems-level debugging in C.

This project progresses through four structured versions, each introducing key features in a simulated garbage collection system:

  • Version 1 – Implements first-fit and best-fit memory allocation strategies with visual memory dumps.
    Also includes: Output (FIRST) | Output (BEST)
  • Version 2 – Introduces the managed pointer system with an indirection layer.
    Also includes: Output
  • Version 3 – Adds generational garbage collection with a copying collector for young generation heaps.
    Also includes: Output
  • Version 4 – Implements old generation with in-place memory compaction and survival tracking.
    Also includes: Output

  • Language: C
  • Compiler: GCC
  • Version Control: Git, GitHub
  • Development Environment: VS Code, WSL
  • Debugging Tools: Valgrind
  • Memory Management Concepts: Heap allocation, minor and major garbage collection, young and old generation heaps
